## Hostile import and version policy

- [ ] 010-C15 Lazy ZIP envelope rejects corrupt/truncated/encrypted/multi-disk/oversized central directory before extraction and never invokes shell/system extractors.
- [ ] 010-C16 Path policy rejects absolute/drive/UNC/backslash/NUL/empty/dot/traversal, non-NFC/case-fold collisions, duplicate names, symlink/device/non-regular/executable/nested-archive and unlisted path/kind fixtures before product writes.
- [ ] 010-C17 Exact `ArchivePolicy/v1` boundary/over-boundary tests enforce 8 GiB compressed upload, 20,000 entries, 240 UTF-8-byte names, 8 MiB manifest, 16 MiB JSON document, 2 GiB media/PDF entry, 16 GiB aggregate expansion and 200:1 per-entry/aggregate ratios with overflow-safe streaming counters; archive/request input cannot raise a cap, the generated managed root/reservation stays 0700/0600, and every failure cleans only its opaque recognized staging reservation—never the external source.
- [ ] 010-C18 Disk preflight formula is boundary-tested before extraction and commit, preserves configured reserve, rejects integer overflow/late free-space loss, and leaves zero visible/import residue on failure.
- [ ] 010-C19 Frozen `{ format: "HekayatiArchive", schemaVersion: 1 }` pure migration produces strict v2 with unchanged external-source checksum; unknown older and every future manifest version reject with a stable bounded message before writes.
- [ ] 010-C20 Every participant domain schema migrates in declared order in staging; missing/future/invalid migration, unknown keys or post-migration reference mismatch rejects the entire import.
- [ ] 010-C21 Checksums, byte counts, canonical JSON, secret absence, referential closure, mode eligibility, media magic/decode and PDF/ICC/template owner facts all validate before `ImportPlan` becomes confirmable.

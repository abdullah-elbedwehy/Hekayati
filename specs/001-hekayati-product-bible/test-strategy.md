# Test Strategy

**Feature**: `001-hekayati` | Constitution XI: behavior-focused, failure-injection first-class, 80%+ coverage on domain/scheduler/adapters/PDF pipeline. TDD during implementation (red→green→refactor).

## Levels

### 1. Unit (Vitest)

Pure logic: customer/consent schemas and both stable refusal codes, including description-only vs photo-derived sheet lineage; family scope, exact name normalization, assign-once anchor, and duplicate-candidate policy; immutable character/look version append + head-CAS; exact 003-owned IM-01–03/05/21 multi-row classification plus 004-owned IM-04 project-override classification; photo compressed/pixel limits, versioned metric thresholds, and warning codes; mention parsing/resolution (Arabic diacritics, spaces, duplicates — FR-040), invalidation engine vs matrix rows (every IM row = at least one test), version lineage + commit preconditions, prompt compiler (negative constraints always present, deny-list FR-071), reference budgeting (capability matrix §4), narration/dialogue balance suggestion; versioned layout-policy candidate order, quietness/contrast scoring and deterministic ties; auto/top/bottom/right/left region math, 14pt/12pt floors, aid order, overflow/long-name/no-safe-area cases, hostile markup/bidi controls, speaker-anchor vs non-pointing fallback; closed source selection and cover warning/readiness; exact printer compatibility; field-membership tests for pageContent/composition/layout/customerContent/approvalBundle/contentAuthorization/derivative hashes, proving operational revisions/status/attention/watermark exclusions; action-ledger hashing; spine-block, idempotency and secret/ZIP rules.

### 2. Contract (Vitest)

Provider adapters tested against `contracts/provider-contract.md` + `structured-outputs.md`:

- **Mock**: full conformance (reference implementation).
- **Codex/Gemini**: same conformance suite behind env-gated live flags + recorded-fixture mode for CI; error-normalization table verified with synthesized provider errors (each taxonomy row → one test).
- **Image boundary**: adapters receive only ephemeral `ResolvedImageRequest` bytes/safe metadata; the harness proves they cannot load arbitrary assets/originals and that drafts, intake tokens, and image bytes never enter logs.
- Schema suite: valid/invalid fixtures per schema (page-count mismatch, alien characterRef, missing negative constraints, deny-list hits).

### 3. Integration (Vitest, real SQLite + real FS in tmp)

Scheduler: atomic multi-job DAG validation (missing/self/cycle/cross-scope), strict provider/local/human request-target union and local descriptor hash resolution, priority/FIFO and fan-out concurrency caps, unique intent/request idempotency collision handling, lease expiry/reclaim with boot + claim-token fencing (including same-worker reclaim), duplicate enqueue, exhaustive retry schedules, provider-wide quota incidents with per-scope wait/successor and later availability-resume decisions, ordinary global provider/model/tier Settings impact+successor transactions, reason-specific pause/resume, cancel-with-late-result, human-gate owner-only positive completion plus negative-outcome cancel/supersede with descendants blocked, persistent storage pause/probe, restart recovery (kill worker process mid-job), and monotonic-clock lease/stall behavior under simulated wall-clock jumps (EC-E05). Provider pre-dispatch fixtures recheck current consent and reference/sheet lineage before capability/network, repeat the guard, and prove rejected paths load zero bytes and make zero adapter/network call; local fixtures prove zero provider capability/consent/adapter/incident path.
Asset store: atomic write crash simulation (kill between temp-write and rename → temp orphan swept; between rename and DB commit → unindexed rename swept, then an explicit retry creates one indexed asset and subsequent identical retries increment its refcount), dedup on identical bytes with canonical-metadata conflict rejection, integrity scan. A per-data-root process lock is acquired before destructive startup recovery so a second launch cannot sweep an in-flight first process's rename. Unowned non-empty data roots fail before mutation, and unknown filenames inside an owned assets directory survive GC.
Repositories: document validation, settings v1→v2 migration, atomic restart-safe Project v1→v2 + PageLayoutHead migration with partial/duplicate rollback, insert-only versions/change events, compare-and-swap head success/stale/duplicate/rollback, and permanent-delete cascade with media removal verification.
Layout/preview/approval: migrate Project/Page heads; derive story layout from exact approved PageReview and special layout from exact policy/selection with forbidden cross-kind review fields; prove lock-only observation revision retries a running job but changes no committed customerContentHash; cover blocking warnings prevent pdf_pending. Pin text/review/source/derivative evidence; validate page boxes, promote file before atomically indexing PreviewOutput + ready cycle + gate. Separate preview/cycle from content-approval heads: same-content IM-19 rerender preserves approved/print-ready; only approval advances authorization; same-content changes-requested revision-checks and revokes it. Drive real emitter/participant/replay coverage for IM-06/08/09/11/12/13/19/20. Approved-book reader tests initial block→zero job, later invalidation→existing job stale/no artifact, pre-execution+commit rechecks, IM-19 during print→same contentAuthorizationHash/commit allowed, referenced IM-20 corruption→block then byte-identical repair/reverification→same authorization restored, and different-byte repair→new version/reapproval.
Reference-photo intake (synthetic people only): real JPEG/PNG plus runtime macOS HEIC conversion; spoofed extension, unsupported/corrupt/truncated, over-byte and over-pixel files; orientation application and independent GPS/EXIF/IPTC/XMP removal checks; private-original vs working/thumbnail/required-face-crop linkage; character- and look-owned fixtures; opaque reservation/token expiry; atomic staged photo-only character creation; duplicate open-existing/create-separate; keyboard boxes for single- and multi-person faces; versioned metric/threshold evidence; provider-reference resolver refusal of originals/full-frame face assets; prepared-write cancel/rollback and child-process kill before metadata commit; startup GC/retry; original-namespace integrity, permissions, symlink, and unknown-file preservation. Every failure asserts no visible `ReferencePhoto`/new character, no advanced owning character/look head, and no ownerless indexed asset.
Local HTTP trust boundary: prove each forbidden listener address is rejected before socket open, then independently verify the accepted listener's effective address after listen; drive a raw HTTP harness across canonical/invalid authority, DNS-rebinding names, forwarded-host spoofing, cross-origin CORS/PNA preflights, source-header variants, every unsafe method, and current/stale CSRF tokens. Each rejection asserts that a persisted sentinel and route-dispatch counter are unchanged (FR-147, FR-148, SC-014).
Portability/deletion: frozen participant completeness catalog and direct-SQL-writer inventory; exhaustive customer/project/template lock hierarchy and drain/snapshot/exclusive admission at every domain mutation plus scheduler enqueue/claim/promote/resume/run/commit point; synchronous-transaction thenable rejection; exact canonical snapshot rows/media holds under concurrent mutation and shared-media release; deterministic `HekayatiArchive/v2` export, frozen v1 migration, two-pass secret gate, and managed download. FR-160 action-ledger tests cover export pause/start, import upload/plan/commit/replace, and deletion confirm/cleanup retry: exact replay before/after restart returns the same bounded result, key/hash collision mutates nothing, upload verifies declared checksum/bytes, and injected transaction failures create no duplicate archive/operation/plan/graph/refcount delta/unlink/report. Every EC-G01–G13 case has a named bounded fixture covering envelope, Unicode/path/type/resource/disk/version/schema/reference/media/secret/mode failures. Import exercises immutable paged plans, collision-heavy typed ID rewrite + derived-hash rebase, all four modes, prepared files, one-transaction visibility, replace releases, and rollback. Project/customer deletion exercises fresh inventory, forced job/gate cancellation, late commit rejection, reverse dependency/refcount changes, shared preservation, exact managed unlink ledgers, post-delete verification, and no out-of-scope mutation. Current real 003–009 data round-trips; a synthetic 011 participant proves registration/omission until slice 011 repeats the real Studio delta.

### 4. Failure injection (dedicated suite)

Scripted mock-provider faults per taxonomy row × retry-policy assertion; ENOSPC/EACCES/EROFS through injected real-FS/asset hooks plus a bounded dedicated-volume fixture where available; SQLite busy/unavailable; network loss; and app-kill matrix: {claimed, provider-running, prepared temp, renamed-before-DB, retry delay, waiting review, story gen, image fan-out, layout render, preview PDF render/validate, portability lock/drain, snapshot freeze/media staging, import validation/prepare/DB commit/cleanup, deletion DB commit/unlink/verification} × restart → SC-002 assertions (no duplicates, completed intact, fences/pauses preserved, old or one complete new/deleted graph only). Preview coverage includes stale/canceled/old-fence return, kill before/after rename and before DB commit, hard-size failure, missing font/watermark, and orphan compensation; no partial file may become current/sendable. Portability recovery never expires a lock by time, deletes an external/operator file, loses a media hold, exposes a partial import, resurrects deleted data, or marks cleanup complete with a failed DB/file/refcount check.

### 5. E2E (Playwright, Arabic RTL UI)

Journeys mirror user stories US1–US11 using the mock provider; assertions on UI state, persisted state, and produced files. US1 runs with no provider. Exact-preview journey covers automatic layout/default source/cover readiness, source edit, sent, approval-only gate success, stale-tab/ledger replay, lock-only no-content-change, punctuation IM-07, IM-18 no-op, IM-19 replacement cycle without lifecycle/authorization regression, same-content changes-requested revocation, and IM-20 exact-repair recovery. It proves materialization-time zero job versus later stale existing job/no artifact, along with three-width UI, secret scans, and origin/CSRF restart behavior.

### 6. Visual regression & golden files

- **Arabic shaping corpus** (gate G3): connected forms, lam-alef, tashkeel, Arabic punctuation, mixed-direction lines (Latin names/numbers), long names — rendered to PDF, rasterized, pixel-diffed against approved goldens (SC-008).
- **PDF geometry goldens**: every preview MediaBox/applicable TrimBox and portrait orientation against the pinned A4/tolerance profile, exact 16/24 interior page order plus front/back preview proofs, and later A4+bleed/crop/cover-spread geometry per test printer profile; watermark presence/absence.
- **Layout presets**: each placement mode + gradient/panel aids over fixture artwork; dialogue anchor/fallback; title/dedication/story/farewell/brand and cover-proof templates; RTL screenshots at 390×844, 1440×900 and 1920×1080 (SC-012); reduced-motion respected.

### 7. Preflight defect fixtures (SC-006)

One seeded defect fixture per FR-123 category (wrong dims, wrong count, missing image, low-res, overflow, missing font, missing bleed, unsafe margins, invalid spread, unknown spine, corrupt PDF, failed conversion, watermark-in-print, missing-watermark-in-preview) — preflight must flag each; suite fails if any fixture passes preflight.

### 8. Privacy & security suite

Secret-scan over: DB dump, full log corpus after a scripted "noisy" run, every export fixture (SC-005), produced preview PDFs, and runtime-token canaries (FR-148). Log-redaction unit tests (Gemini key patterns, image bytes). File-permission assertions (0700/0600), including the private original namespace. Bind-address tests cover EC-H06 plus every FR-147 listener/authority fixture. Browser-request forgery tests cover EC-H09–H13 and CHK222–226, including `no-store` bootstrap assertions, without relying on browser CORS enforcement alone. Consent tests cover absent/refused/revoked-before-dispatch with zero adapter/network calls for direct photos and photo-derived sheets, plus successful no-consent description-only and wholly description-derived-sheet fixtures (EC-H01/H14). Upload tests prove bounded parts/files, generated private temp names, opaque non-URL intake tokens, no client path leakage, no image bytes/metadata in logs, originals unreachable through ordinary browser/provider APIs, and zero external analysis calls. Template-from-story privacy stripping fixture (FR-051): source story with photos/names → template contains neither. Preview fixtures inject HTML/script, remote CSS/image/font, `file://`, unsafe bidi controls, internal IDs/contact/consent text and full-resolution source bytes; rendering escapes text, blocks every external/file request, strips metadata/attachments/scripts/forms, embeds only approved downsampled derivatives/fonts, and records zero egress.

## Live-provider validation (manual, gated, not CI)

Phase 0 gate scripts G1-T/G1-I/G2/G3/G4 with structured scorecards recorded into research.md. Recurring smoke: one cheap Gemini structured call + one image call behind an operator-triggered "connection test" (FR-105) — never automatic in CI.

## Coverage & gates

- Coverage floor 80% on `domain/`, `jobs/`, `providers/`, `pdf/`, `portability/`, `security/`, and `server/security/` (per-directory, not just global).
- CI order: lint → typecheck → unit → contract(mock+fixtures) → integration → failure-injection → E2E(mock) → golden/visual → privacy suite.
- Every FR referenced by at least one test ID or checklist item (traceability audited at the analyze stage; checklists cross-link).
- AAA structure; descriptive behavior names; no implementation-detail assertions (Constitution XI).
